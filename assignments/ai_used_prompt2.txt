설명: (c)는 캡처 그룹이 없어 일치하는 전체 문자열을 반환하고, (d)는 캡처 그룹 ()을 사용하여 추출된 텍스트들을 튜플로 묶어 반환합니다. 반면 (e)는 비캡처 그룹 (?:)을 사용하여 패턴을 논리적으로 묶기만 하고 별도로 캡처하지 않으므로 (c)와 동일하게 전체 매칭 문자열을 돌려줍니다.  

[각 단계별 상세 해설]

(a) re.match(r"\d+", text): match 함수는 무조건 문자열의 맨 처음부터 일치하는지만 검사합니다. 텍스트가 한글인 "강의는"으로 시작하므로 숫자(\d+) 패턴과 맞지 않아 매칭에 실패하고 None을 반환합니다.

(b) re.search(r"\d{4}-\d{2}-\d{2}", text).group(): search 함수는 문자열 전체를 뒤져서 가장 먼저 일치하는 부분을 찾습니다. 첫 번째 날짜인 "2026-05-06"을 찾고, .group()을 호출하여 해당 문자열을 그대로 반환합니다.

(c) re.findall(r"\d{4}-\d{2}-\d{2}", text): findall 함수는 일치하는 모든 패턴을 찾아 리스트로 반환합니다. 패턴 내에 캡처 그룹(소괄호)이 없기 때문에, 일치한 전체 텍스트 2개가 리스트에 담겨 반환됩니다.

(d) re.findall(r"(\d{4})-(\d{2})-(\d{2})", text): 정규표현식에서 괄호 ()는 캡처 그룹을 의미합니다. findall에 캡처 그룹이 포함된 패턴을 넘기면, 동작 방식이 바뀌어 일치하는 전체 문자열 대신 괄호 안에서 매칭된 부분들만 튜플로 묶어서 반환합니다.

(e) re.findall(r"(?:\d{4})-(?:\d{2})-(?:\d{2})", text): 괄호 시작 부분에 ?:를 붙이면 비캡처 그룹이 됩니다. 즉, 패턴을 하나로 묶어주는 역할만 하고 추출 결과물에는 영향을 주지 않겠다는 뜻입니다. 캡처 그룹이 존재하지 않는 것으로 취급되므로 (c)와 동일한 결과가 반환됩니다.

설명 (i): (a)는 탐욕적(Greedy) 수량자인 .+를 사용하여 첫 <부터 문장 끝의 가장 마지막 >까지 한 번에 길게 매칭한 반면, (b)는 게으른(Lazy) 수량자인 .+?를 사용하여 < 이후 처음 만나는 >까지만 짧게 끊어서 매칭했기 때문입니다.   


설명 (ii): (d)는 원시 문자열(r"...")을 사용하여 \1이 정규식의 1번 캡처 그룹(추출된 숫자)으로 올바르게 해석되지만, (e)는 일반 문자열이므로 파이썬이 \1을 아스키코드 1번(SOH, 제어 문자)으로 자체 해석하여 이스케이프 해버렸기 때문입니다.   

[각 단계별 상세 해설]

(a) re.sub(r"<.+>", "[T]", html): +나 * 같은 기본 수량자는 '탐욕적(Greedy)'입니다. 텍스트 <b>안녕</b> <i>세상</i>!에서 맨 앞의 <를 찾은 뒤, 조건(.)을 만족하면서 최대한 뒤에 있는 >까지 다 먹어 치웁니다. 그래서 <b>부터 </i>까지 통째로 하나로 인식되어 [T]로 바뀌고, 뒤에 남은 !만 붙어 '[T]!'가 됩니다.

(b) re.sub(r"<.+?>", "[T]", html): 수량자 뒤에 ?를 붙이면 '게으른(Lazy)' 탐색을 합니다. <를 찾은 뒤, 조건 만족을 최소화하여 가장 먼저 만나는 >에서 탐색을 멈춥니다. 따라서 <b>, </b>, <i>, </i> 태그 4개가 각각 개별적으로 인식되어 치환됩니다.

(c) re.sub(r"<[^>]+>", "[T]", html): 정규표현식 내에서 [^>]는 "> 기호가 아닌 모든 문자"를 뜻합니다. <로 시작해서 >가 아닌 문자열들이 오다가 >로 닫히는 패턴을 찾으므로, 결과적으로 (b)와 동일하게 각각의 HTML 태그만 정확히 짚어내어 치환합니다. 실무에서 태그를 지울 때 (b)보다 이 방식을 더 자주 씁니다.

(d) re.sub(r" (\d+)", r"<\1>", nums): 치환할 문자열에 원시 문자열 기호 r을 붙였습니다. 이 덕분에 백슬래시(\)가 파이썬의 이스케이프 문자로 작동하지 않고 순수하게 정규표현식 엔진으로 넘어가, 1번 캡처 그룹인 (\d+)(숫자들)을 그대로 불러오는 역할을 완벽하게 수행합니다.

(e) re.sub(r" (\d+)", "<\1>", nums): 치환 문자열에 r이 없습니다. 파이썬은 문자열 안의 \1을 보고 정규표현식 그룹 번호가 아니라 "아, 8진수 1번 아스키코드(제어 문자)를 쓰라는 거구나!"라고 오해합니다. 그 결과 추출한 숫자가 들어가야 할 자리에 요상한 특수문자(\x01)가 들어가 버리게 됩니다. 정규식을 쓸 때 항상 r""을 습관화해야 하는 이유입니다.

import re
from collections import Counter

# 동일한 패턴이 반복 사용되므로 미리 컴파일 (성능 최적화)
pat_url = re.compile(r"https?://\S+")
pat_html = re.compile(r"<[^>]+>")
pat_email = re.compile(r"[\w.-]+@[\w.-]+\.[a-zA-Z]{2,}")
pat_phone = re.compile(r"\d{2,4}-\d{3,4}-\d{4}")
pat_mention = re.compile(r"@\w+")
pat_hashtag = re.compile(r"#\w+")
pat_kr_con_vow = re.compile(r"[ㄱ-ㅣ]+")
pat_ws = re.compile(r"\s+")

# 해시태그 추출용 (괄호로 캡처 그룹 지정)
pat_hashtag_ext = re.compile(r"#(\w+)")

def clean_post(post: str) -> str:
    # 6단계 정제 파이프라인
    text = pat_url.sub(" ", post)
    text = pat_html.sub("", text)
    text = pat_email.sub("[이메일]", text)
    text = pat_phone.sub("[전화]", text)
    text = pat_mention.sub(" ", text)
    text = pat_hashtag.sub(" ", text)
    text = pat_kr_con_vow.sub("", text)
    text = pat_ws.sub(" ", text).strip()
    return text

def extract_hashtags(post: str) -> list[str]:
    # 캡처 그룹(\w+)을 사용했으므로 #을 제외한 알맹이만 리스트로 반환됨
    return pat_hashtag_ext.findall(post)

def analyze_posts(posts: list[str]) -> dict:
    total_masked = 0
    all_hashtags = []
    cleaned_lengths = []

    for post in posts:
        # 해시태그 추출 (원본에서)
        all_hashtags.extend(extract_hashtags(post))

        # 마스킹 횟수 계산을 위해 1, 2단계만 적용한 임시 텍스트 생성
        temp = pat_url.sub(" ", post)
        temp = pat_html.sub("", temp)
        
        # re.subn을 활용하여 마스킹된 치환 횟수 누적
        temp, email_cnt = pat_email.subn("[이메일]", temp)
        temp, phone_cnt = pat_phone.subn("[전화]", temp)
        total_masked += (email_cnt + phone_cnt)

        # 정제 및 글자 수 계산
        cleaned = clean_post(post)
        cleaned_lengths.append(len(cleaned))

    # 평균 길이 계산 (소수점 2자리 반올림)
    avg_len = round(sum(cleaned_lengths) / len(posts), 2) if posts else 0.0

    return {
        "posts_n": len(posts),
        "avg_length_after_clean": avg_len,
        "hashtag_counts": dict(Counter(all_hashtags).most_common()),
        "masked_count": total_masked
    }

# ---------------- 테스트 실행부 ----------------
posts = [
    "오늘 #파이썬 수업 진짜 재밌었음!! @prof_kim @hong 감사 ㅎㅎ",
    "자료: https://etl.snu.ac.kr/lec17",

    "<b>중요</b>: 다음 시험 범위는 1-15강.\n문의는 mam3b@snu.ac.kr (010-1234-5678)로!",
    "\n\n\n여러 공백과\n\n\n줄바꿈이 많은   텍스트",
    "ㅋㅋㅋ #파이썬 진짜 좋다 #추천 https://snu.ac.kr"
]

print("(1) 각 게시물에 대한 clean_post 반환값:")
for p in posts:
    print(repr(clean_post(p)))

print("\n(2) analyze_posts 반환 딕셔너리:")
print(analyze_posts(posts))

💡 [단계별 상세 해설]
(a) clean_post 6단계 파이프라인의 원리:

URL (https?://\S+): 웹 주소는 가장 길고 구조가 복잡하므로 제일 먼저 덜어내야 다른 정규식과 충돌하지 않습니다.

HTML (<[^>]+>): 태그를 비워("") 글자들만 남깁니다. 여기서 남은 텍스트들 안에서 본격적인 개인정보 탐색을 시작합니다.

이메일/전화번호 마스킹: 가장 핵심입니다. 이메일 중간에 들어간 @ 기호가 뒤에 올 '멘션' 처리 단계에서 오작동을 일으키지 않도록 개인정보부터 안전하게 덮어([이메일]) 씌워야 합니다.

멘션/해시태그 (@\w+, #\w+): 이제 안심하고 소셜미디어 특수 기능인 태그들을 지웁니다. 힌트에서 언급됐듯 파이썬의 \w는 한글을 포함하기 때문에 한국어 해시태그도 완벽히 잡아냅니다.

자음/모음 ([ㄱ-ㅣ]+): ㅋㅋ, ㅎㅎ, ㅠㅠ 같이 문맥에 큰 의미가 없는 초성/중성 감탄사를 제거합니다.

공백 정리 (\s+): 앞선 단계들에서 치환하면서 남겨진 잉여 공백들이나 원래 있던 줄바꿈(\n)들을 모두 깔끔하게 띄어쓰기 한 칸(" ")으로 압축하고, 양끝 공백(strip())을 날립니다.

(b) extract_hashtags 함수:
단순히 #\w+로 찾으면 #파이썬처럼 샵 기호까지 같이 나오게 됩니다. 하지만 우리가 작성한 re.compile(r"#(\w+)")처럼 괄호(캡처 그룹)를 씌워주면, 파이썬이 알아서 #은 매칭 조건으로만 쓰고 괄호 안의 알맹이(파이썬)만 추출해 줍니다.

(c) analyze_posts 내 치환 횟수 산출 (re.subn):
단순히 문자열 개수를 세는 .count()를 쓰면 원래 본문에 우연히 적혀있던 [이메일]이라는 글자까지 세어버릴 위험이 있습니다. re.subn()을 사용하면 "정규식이 실제로 작동해서 치환을 수행한 횟수"를 (바뀐문자열, 치환횟수) 형태의 튜플로 정확히 돌려주기 때문에 훨씬 안전하고 프로다운 방법입니다.
